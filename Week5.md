# 학습 내용 정리

## 39강. LOD

<!-- INCLUDE, EXCLUDE, FIXED 등 본 강의에서 알게 된 LOD 표현식에 대해 알게 된 점을 적어주세요. -->

- LOD(Level of Detail) 표현식은 시각화의 표시 수준과 다른 수준의 집계값을 계산할 수 있도록 해주는 기능이다.
- 대표적인 표현식:
{FIXED [차원]: 집계함수}
{INCLUDE [차원]: 집계함수}
{EXCLUDE [차원]: 집계함수}
- LOD 표현식을 사용하면 시각화에서 선택된 차원과 무관하게 특정 차원 기준으로 정확한 값을 제어할 수 있다.

## 40. LOD EXCLUDE

<!-- INCLUDE, EXCLUDE, FIXED 등 본 강의에서 알게 된 LOD 표현식에 대해 알게 된 점을 적고, 아래 두 질문에 답해보세요 :) -->

- {EXCLUDE [차원]: 집계함수}는 현재 시각화에 포함된 차원 중 특정 차원을 제외한 상태로 집계한다.

> **🧞‍♀️ FIXED와 EXCLUDE을 사용하는 경우의 차이가 무엇인가요?**

```
- FIXED는 시각화와 무관하게 **지정된 차원 기준으로 고정**된 집계를 수행한다.
- EXCLUDE는 시각화에 포함된 차원 중에서 특정 차원을 **제외하고 계산**한다.
- 예를 들어, 시트에 [고객]과 [카테고리]가 있을 때:
  - `{FIXED [고객]: SUM([판매액])}` → 고객 기준으로 고정
  - `{EXCLUDE [카테고리]: SUM([판매액])}` → 카테고리를 제외하고 나머지 차원 기준으로 집계
```

> **🧞‍♀️ 왜 ATTR 함수를 사용하나요?**

```
- ATTR은 특정 필드가 다중 값일 수 있는 경우 "단일 값인지 여부"를 판단해주는 함수다.
- 시각화에 여러 값이 존재하는 상황에서도 에러 없이 데이터를 표시하거나,
  단일 값일 경우만 제대로 표시되도록 안전장치 역할을 한다.
- 특히 텍스트 마크나 툴팁에서 차원이 혼합되어 있을 때 유용하다.
```



## 41. LOD INCLUDE

<!-- INCLUDE, EXCLUDE, FIXED 등 본 강의에서 알게 된 LOD 표현식에 대해 알게 된 점을 적고, 아래 두 질문에 답해보세요 :) -->

- {INCLUDE [차원]: 집계함수}는 시각화에 추가 차원을 포함하여 더 세분화된 집계값을 계산한다.

> **🧞‍♀️ 그렇다면 어떤 경우에 각 표현식을 사용하나요? 예시와 함께 적어보아요**


```
- FIXED: 특정 기준으로 값을 고정할 때 (예: 고객별 총매출 → {FIXED [고객]: SUM([매출])})
- INCLUDE: 더 상세한 수준의 값을 계산한 후 상위 수준으로 집계할 때 (예: 주문 항목별 매출 → 고객 수준으로 평균)
- EXCLUDE: 과도하게 세분화된 시각화에서 특정 차원을 제외하고 집계할 때 (예: 날짜별 데이터를 제외한 월별 합계 계산)
```



## 42. 매개변수

<!-- 매개변수에 대해 알게 된 점을 적어주세요 -->

- 매개변수(Parameter)는 사용자가 동적으로 값을 입력할 수 있는 컨트롤 요소이다.
- 필터, 계산된 필드, 집합 등 다양한 곳에 연결해 동적 시각화를 구현할 수 있다.


> **🧞‍♀️ 집합에도 매개변수를 적용할 수 있나요? 시도해봅시다**



## 43. 매개변수 실습

<!-- 영상 묶음에 포함되지 않아 찾기 어려우실까 링크를 아래에 첨부하겠습니다. 수강 후 삭제해주세요-->

- 매개변수를 숫자 슬라이더, 드롭다운, 입력 필드 등으로 구현해 다양한 필터링과 동적 조건부 분석이 가능하다는 점을 실습을 통해 체험할 수 있었다.


## 44. 매개변수 실습

<!-- 매개변수에 대해 알게 된 점을 적어주세요 -->

- 매개변수는 단순 필터가 아닌, 계산된 필드와 결합하여 더 정교한 조건문 제어에도 활용 가능하다.
- 사용자에게 그래프 유형, 기준 지표, 조건 범위 등을 선택권으로 부여할 수 있어 대시보드의 인터랙션을 높일 수 있다.

## 46. 서식 계층
